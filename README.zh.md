## 📖 缘起

随着AI科技日新月异，AI 立法正在全球范围内加速。在美国加州州众议员实习期间，团队队长 Amanda 亲历 AI 立法的一线挑战，发现了人民生成法案的缝隙：

|  | 问题 | 现状 |
|--|------|------|
| 一 | 政府无从下手 | AI 立法没有现成剧本 |
| 二 | 法案语言是壁 | 普通人根本读不懂条文 |
| 三 | 跨法域信息孤岛 | 市面上缺乏跨法域对比工具 |
| 四 | 被动立法而非主动 | 危机后政府才去立法 |
| 五 | AI 立法不能只靠内部人 | 受影响的人无法发声 |

如果以夷制夷，用AI 为 AI 立法，能否可行？

为了寻找答案，我做了一个小律提案。

## ✨ 核心亮点

### 1. 人机双轨立法 —— Human + Autonomous 双模式

**人工引导链：**
搜索既有法案 → 生成草案 → 分析通过率 → 社媒活动推广，形成完整的立法倡导闭环。每个 Agent 既可独立使用，也可串联协作。

**纯机器的 AI黑盒：**
全球首个 24/7 不间断的自主 AI 立法机。无需任何人干预，持续生成 AI 政策法案。

### 2. 跨法域智能 —— 三地数据融合

同时利用香港（行政主导）、澳门（行政长官事前同意）、美国加州（两党制）三种截然不同的制度环境数据进行检索和生成，形成独特的"制度光谱"研究框架。

### 3. 法案起草透明化

让普通人也能参与立法过程——搜索先例、生成草案、分析成功率、发起请愿、社媒活动推广，每一步都有 AI 辅助。

---
## 🎬 完整功能巡礼

从个人诉求到全网联署，小律提案构建了一条完整的立法参与链。

### 第一步：搜索 —— 跨法域检索先例

在仪表盘搜索功能中输入主题，即可检索到过去相关的法案。随后使用 **搜索 Agent** 进行更深入的跨法域搜索——可选择专注特定法域（香港/澳门/美国加州），也可一键搜索全部法域。系统精准返回对应法案总数与详细列表。

### 第二步：生成 —— AI 起草法案 + 议员匹配 + 请愿

**生成 Agent** 让你输入主题即可起草专业法案。系统提供三种量身定制的字数模式——**标准版**、**详细版**与**简洁版**。选择模式点击生成，片刻之间，一份格式严密、条文完整的法案便呈现在面前。

不仅如此，它还内置两个核心 Subagent：

- **议员匹配 Subagent**：自动检索历史提案数据，精准匹配曾支持过类似法条且可能支持你提案的立法会议员
- **请愿 Subagent**：一键发起联署收集公众签名，实时显示最新签名动态

从法案撰写、议员游说到民意收集，全链路闭环一气呵成。

### 第三步：分析 —— 通过率定量模拟

**分析 Agent** 对法案进行深度的定量模拟，内置三个核心 Subagent：

- **画图分析 Subagent**：自动生成直观的预测图表，清晰展示各党派对该主题的预测支持率
- **党派分析 Subagent**：逐一剖析每个独立党派的投票倾向与立场拆解
- **总体通过率 Subagent**：综合评估法案的总体通过率，并同步输出详细分析报告，一针见血指出法案的核心优势与潜在阻力

### 第四步：社媒活动 —— 影响力扩散

**社媒活动 Agent** 自动设定联署签名目标，内置六种不同的文案角度模板。通过注入特定平台语调指南的系统提示词，AI 能够一键生成适配不同社交平台的推广内容——无论是以视觉为主的 Instagram，还是带有强种草属性的小红书。至此，从个人诉求到全网市民联动的完整闭环完美达成。

### 第五步：AI 黑盒 —— 24/7 自主立法

最后来到整个平台最核心的引擎：**AI黑盒**。在这里，AI 驱动的自主立法流 24 小时不知疲倦地自动感知前沿技术漏洞，持续生成全新的法案草案——完全无需任何人类输入。系统持续自主演进并完整生成新法案，伴随新的时间戳实时接入系统。

**综上所诉：**

| Agent | 角色 | 输入 → 输出 |
|-------|------|------------|
| 🔍 搜索 | 法案检索 | 自然语言查询 → 相似法案列表（含相关性评分） |
| ✎ 生成 | 法案起草 | 主题 + 风格 → 完整法案草案（含议员推荐 + 请愿） |
| 📊 分析 | 通过率预测 | 法案主题 → 通过率 + 政党倾向 + 图表 |
| 📣 社媒活动 | 社媒活动推广 | 法案信息 → 多平台适配帖子（含请愿 CTA） |
| 📦 AI黑盒 | 自主立法 | 无输入 → 不间断生成 AI 法案（随机主题/法域） |

## 🤖 智能体融合

### 多智能体协同机制

每个 Agent 独立调用 Deepseek 大模型，通过结构化 Prompt 工程实现专业分工：

- **搜索 Agent**：利用 384 维向量嵌入进行语义检索，支持中英文混合查询，中文自动翻译为英文后再检索
- **生成 Agent**：三段式 Prompt（角色设定 + 任务描述 + 格式约束），支持标准/详细/简洁三种输出粒度
- **分析 Agent**：三段定量模拟（画图分析 + 党派分析 + 总体通过率），结构化输出预测图表与评估报告
- **社媒活动 Agent**：内置六大文案模板角度（问题-解决方案、热点话题、故事分享、社区互动、进展更新、紧急呼吁），平台风格指南作为 System Prompt 注入
- **AI黑盒**：25 个 AI 政策主题池随机轮换，3-6 秒间隔持续调用，实现无监督自主生成

### 模型优化

- 通过精确的 Prompt Engineering（角色设定、输出格式约束、语言强制指令）减少幻觉和格式错误
- 语言隔离策略：中英文模式下分别追加强制语言指令，杜绝语言混杂
- 使用 Vercel 无服务器部署，冷启动优化，API 超时 90-120 秒适配长文本生成

## 🛠️ 技术栈

| 分层 | 技术 | 说明 |
|--------|------|------|
| 后端框架 | Python / FastAPI | 轻量异步 Web 框架 |
| 前端 | 静态 HTML + Vanilla JS | 零依赖，直出渲染 |
| AI 推理 | Deepseek（Anthropic 兼容 API）+ StepFun | 社媒活动 Agent 与 AI黑盒由 StepFun 驱动；搜索、生成与分析 Agent 接入 Deepseek。双引擎联手 NVIDIA DGX Spark，实现全面自动化 |
| 向量检索 | NumPy + cosine similarity | 384 维嵌入语义搜索 |
| 部署平台 | Vercel（`@vercel/python`） | 无服务器快速部署 |
| 数据 | 三法域立法语料库 | 香港（LegCo）+ 澳门 + 美国加州（leginfo） |

### 使用的 NVIDIA 及第三方工具

- **NVIDIA SDK**：DGX Spark 平台全栈能力
- **Stepfun 阶跃星辰**：配套大模型支持
- **Deepseek v4**：通过 Anthropic 兼容接口驱动 AI 推理
